Zork I — v2: Bug Fixes

The First Version That Changes the Game

Play Play in IF Hub Source Walkthrough

v2 is where improvement begins. Only here do we allow ourselves to actually fix things that were bugs in the original ZIL. This is also where the testing methodology was built — deterministic seed-based walkthroughs, side-by-side ZIL vs. Inform 7 transcript diffing, and automated regression testing.

Bugs Fixed

  • Deflating the boat lost its contents — a “Raft of Holding” bug (#2)
  • Putting the raft in the coffin and the coffin in the raft caused both to vanish (#3)
  • Phantom combat — you could attack enemies after they left the room (#6)
  • Rising water killed the player even inside the boat (#15)
  • Grate could be locked while open (#16)
  • Sharp objects puncturing the boat also destroyed its contents (#22)
  • Lit match survived death and resurrection (#11)
  • Weapon penalty applied even on combat misses (#4)
  • 12 behavioral differences found via transcript comparison (#63)

Testing Innovation

v2 introduced the transcript comparison methodology that now drives all testing. Compile both the ZIL original and the Inform 7 translation with the same deterministic seed, run identical command sequences, and diff the output line by line. Every divergence either reveals a translation gap or confirms an intentional fix.
